# ADR-001. Редизайн Bookechi — «cozy reading companion»

> Статус: **Proposed** · Дата: 2026-06-09 · Автор: продуктовая сессия с Claude
> Формат: ADR-like (Architecture/Design Decision Record). Один документ — контекст, драйверы,
> зафиксированные решения, открытые вопросы.

---

## 1. Контекст

Есть рабочий прототип Android-приложения для трекинга чтения (Jetpack Compose, MVI, Room).
Функционально каркас собран, но визуально и по UX он выглядит как «generic Material app».
Цель — переосмыслить интерфейс как **уютный reading companion**: тёплая бежево-кремовая палитра,
тёмно-коричневая типографика, оранжевые акценты, мягкие радиусы, много воздуха, чёткая иерархия.

Что **уже есть** в коде (фундамент, на который опираемся):

| Слой | Состояние |
|---|---|
| Навигация | 3 нижних таба: **Активность** (`BookListScreen`), **Продуктивность** (`ProductivityScreen`), **Библиотека** (`LibraryScreen`) |
| Модель `Book` | `name, author, coverPath, pages, currentPage, readingStatus, isFavorite` |
| `ReadingStatus` | `None, Planned, Reading, Paused, Dropped, Completed` (**6 значений**) |
| `ReadingSession` | посуточная статистика: `pagesRead, readingTimeMinutes, startPage, endPage` |
| Палитра | в `Color.kt` уже оцифрован тёплый язык под префиксом `Figma*` (кремовый фон, коричневый текст, оранж, heatmap на 6 уровней, стрик) |
| Тема | ⚠️ `Theme.kt` включает `dynamicColor = true` + фиолетовую dark-схему — **это ломает тёплую палитру** на Android 12+ |
| Типографика | `FontFamily.Default` — кастомных шрифтов нет |

Вывод: **дизайн-язык частично существует**, но не консолидирован в систему и подавляется Material-дефолтами.

---

## 2. Драйверы решений

- **D1.** Тёплый, editorial, «бумажный» визуал — не Material из коробки.
- **D2.** Главный сценарий = открыл → увидел текущую книгу → быстро отметил прогресс. Одна активная книга в фокусе.
- **D3.** Единая дизайн-система и переиспользуемые компоненты (а не разнородные экраны).
- **D4.** Все состояния, включая пустые/zero, должны быть спроектированы.
- **D5.** Готовность к дальнейшей разработке: токены и компоненты ложатся на текущую модель данных.

## 3. Зафиксированные решения

### ADR-3.1 — Навигация: 3 таба + модальные потоки
Сохраняем структуру из 3 нижних вкладок и оставляем «тяжёлые» действия отдельными экранами/боттом-шитами:
- Табы: **Активность (Home)** · **Продуктивность (Stats)** · **Библиотека (Library)**.
- Потоки поверх: **Добавить книгу**, **Редактировать книгу**, **Отметить прогресс**, **Успех**.
- **Следствие:** 10 требуемых экранов раскладываются на 3 таба × состояния + 4 flow-экрана.

### ADR-3.2 — «Активная книга» как центральная сущность Home
На Home в фокусе ровно одна активная книга (hero-card).
- Активной считается книга со статусом `Reading`.
- Если их несколько — активной показываем ту, по которой **последний `ReadingSession`** (последний прогресс).
- **Следствие:** нужна детерминированная выборка активной книги; на будущее — возможность «закрепить» книгу вручную (см. Открытые вопросы).

### ADR-3.3 — Статусы: в дизайне показываем 3 + признак «избранное»
В UI оперируем тремя статусами — **Читаю / В планах / Прочитано** — плюс независимый флаг **Избранное**.
- `Paused` и `Dropped` в модели **оставляем**, но **в дизайн v1 не выносим** (вне primary-сценария).
- Маппинг для отображения: `Reading→Читаю`, `Planned→В планах`, `Completed→Прочитано`, `None`→трактуем как «В планах» по умолчанию при создании.
- **Следствие:** фильтры библиотеки = `Все · В планах · Читаю · Прочитано · Любимое` (ровно по требованиям).

### ADR-3.4 — Прогресс: посуточные сессии, страницы — primary, время — опционально
Единица учёта — `ReadingSession` за день по книге.
- Пользователь вводит **страницу, на которой остановился сегодня** (`endPage`). `startPage` = `currentPage` книги на момент старта.
- `pagesRead = endPage − startPage` считается автоматически.
- **Время чтения** — необязательное поле (`readingTimeMinutes`); если заполнено, открывает производные метрики (см. ADR-3.7).
- После сохранения `Book.currentPage = endPage`; при `endPage == pages` предлагаем перевести в `Прочитано`.
- **Следствие:** на экране прогресса — крупный ввод страницы, справа total pages, авторасчёт «прочитано сегодня», валидации.

### ADR-3.5 — Тема: тёплая светлая + тёплая тёмная, без dynamicColor
- **Убираем** `dynamicColor` и фиолетовую `darkColorScheme`.
- **Меняем палитру** на «**Terracotta & Linen**». Причина: текущий акцент `#ED8835`/`#CB4700` слишком насыщённый («строительный оранж») и читается дёшево. Новый акцент — приглушённая терракота `#BE5E3B`/`#9E4A2C`, второй акцент — шалфей `#7C8A6E`, фон — льняной `#F4ECE1`, текст — орех `#382A20`.
- Тёмная тема — **тёплый ореховый** фон (`#1C1611`, не серый Material), акцент-терракота с поднятым контрастом (`#CE6E48`).
- Точные значения токенов (светлая + тёмная, heatmap, стрик) — в `docs/design-prompt.md`.
- **Следствие:** `Figma*`-цвета заменяются новой палитрой и переезжают в именованные семантические токены (`canvas, surface, stroke, textPrimary, accent, secondary…`), а не используются россыпью.

### ADR-3.6 — Дизайн-система: токены + инвентарь компонентов
Единый набор: spacing-шкала (4/8/12/16/20/24/32), радиусы (button 16–20, card 24, hero 28–32, pill/chip — full), типографическая шкала, и переиспользуемые компоненты:
`BookCover (+placeholder)`, `StatusChip`, `FavoriteToggle`, `PrimaryButton`, `SecondaryButton`, `ProgressBar`,
`StreakBadge/StreakStrip`, `MetricCard`, `PeriodSwitcher (месяц/год)`, `Heatmap`, `BookHeroCard`,
`BookListItem (компакт)`, `FilterChips`, `FAB`, `BottomNav`, `EmptyState`, `PageInput`, `WarmTextField`, `GreetingHeader`.
- **Шрифт (принято):** **тёплый serif для заголовков** (Fraunces/Lora/Source Serif) + чистый sans (Inter) для тела. Сейчас `FontFamily.Default` — часть «generic»-проблемы, заменяем.

### ADR-3.7 — Метрики статистики: точные определения
4 карточки + переключатель период (месяц/год) + heatmap + итоговая карточка периода:

| Метрика | Определение за выбранный период |
|---|---|
| Книг прочитано | число книг, перешедших в `Completed` за период |
| Страниц прочитано | `Σ pagesRead` по всем `ReadingSession` периода |
| Дней без перерывов (стрик) | максимальная цепочка подряд идущих дней с `pagesRead > 0` |
| Стр./день в среднем | `Σ pagesRead / число активных дней` (дни с чтением) |

- Heatmap — те же 6 уровней интенсивности (по `pagesRead`), что уже в палитре.
- Производные (**приняты**): **примерная дата дочитывания** активной книги (на hero-card Home, из ср. стр/день); **скорость стр./час** — показываем в Stats только если велось время чтения.

### ADR-3.8 — Каталог состояний (обязательно к отрисовке)
- **Home:** (a) нет книг вообще, (b) книги есть, активной нет, (c) активная есть, но сегодня прогресс не отмечен, (d) активная есть, прогресс отмечен.
- **Stats:** zero-state без данных + наполненное состояние.
- **Library:** empty-state + наполненное.
- **Add/Edit:** обычное + состояние «статус = Читаю» (появляется поле текущей страницы) + ошибки валидации.
- **Progress:** ввод + ошибки валидации.
- **Success:** одно состояние (+ вариант «книга дочитана»).

### ADR-3.9 — Язык и тон
Интерфейс на **русском** (как в текущем прототипе). Тон фидбэка — мотивирующий, но сдержанный; без «геймификационного шума».

## 4. Решённые вопросы и оставшиеся открытые

**Решено:**
- **Палитра** → «Terracotta & Linen» (ADR-3.5).
- **Шрифт** → serif для заголовков + sans для тела (ADR-3.6).
- **Производные метрики** → прогноз дочитывания (Home) + скорость стр/час при наличии времени (Stats) — приняты.

**Осталось открытым (не блокирует дизайн v1):**
- **OQ-2. Закрепление активной книги.** Нужен ли ручной «pin», или автоматики (статус + последний прогресс) достаточно для v1?
- **OQ-3. Paused/Dropped.** Точно прячем из v1-дизайна, или нужен хотя бы статус «отложено» в библиотеке?

## 5. Принятые доп-фичи «от себя»

1. **Прогноз дочитывания** на hero-card: «при твоём темпе ~6 дней до конца».
2. **Финиш книги** на Success-экране: когда `currentPage == pages` — отдельная тёплая «книга прочитана», авто-перевод в `Прочитано`.
3. **Оценка (1–5 ★) + цитата/заметка** при завершении книги (задел под «полку прочитанного»).
4. **Скорость чтения** (стр/час) — показываем только если пользователь вёл время.

## 7. Бэклог v1.0 (после ревью первых макетов)

После генерации макетов и продуктового разбора согласован объём UX-фич. **Дизайн рисуем для всего v1**;
интеграцию Open Library откладываем на код (UI рисуем сейчас).

### В v1.0
| Фича | Суть | Где живёт |
|---|---|---|
| Быстрый лог на Home | Чипы `+10 / +25 / +50 / ⊕` на hero-card; тап → сразу «Успех» | Home |
| Слайдер прогресса | Слайдер вдоль книги + точный ввод | Экран прогресса |
| Режимы ввода | Сегмент `Страницы / Проценты / +N` | Экран прогресса |
| Недельная цель | По **объёму**: «На этой неделе 340/400 стр.» + бар. Раньше были дни — дублировало стрик, поэтому развели смыслы (стрик=дни, цель=объём) | Home + Stats; настройка — шит, по умолчанию «страниц в неделю» |
| Напоминание | Тумблер + выбор времени | Экран настройки в Settings |
| Comeback-состояние | Тёплое «С возвращением» при пропуске, без вины | Home (вариант состояния) |
| Карточка книги (полная) | Обложка, прогресс, история сессий (спарклайн), цитаты/заметки, оценка, CTA | Новый экран; тап по книге в Библиотеке ведёт сюда |
| Инсайт-строка | 1 живая плашка («+18% к маю», «рекорд серии») | Stats |
| Поиск книги | Экран поиска + автозаполнение из каталога | **Дизайн в v1, код позже** (Open Library, без своего сервера) |

### Отложено в будущее
- **Заморозка стрика** (1–2 дня отдыха без обрыва серии) — оверхед для v1.
- **Поиск / сортировка в библиотеке** — не нужно на текущем объёме книг.
- **Интеграция Open Library в код** — UI готовим в v1, подключение API позже.

### Связанные решения
- Режим «Проценты» расширяет ADR-3.4 (прогресс не только по номеру страницы).
- Карточка книги даёт «дом» оценке и цитатам из ADR-5.3 (раньше их некуда было показать).
- Недельная цель и инсайты опираются на метрики ADR-3.7.

## 8. Статус дизайна: утверждён, переходим в разработку (2026-06-10)

После итераций v1 (фичи) + переработки недельной цели и чистки экрана прогресса дизайн
**утверждён как основа для реализации**. Макеты: `claude-design/Скриншоты экранов v1/Скриншоты экранов 2`.

### Что финализировано в макетах
- Недельная цель — по объёму («340/400 стр.», «осталось 60 стр. — примерно один вечер»), не дублирует стрик.
- Экран прогресса — без табов `Проценты/+N`: число + терракотовый слайдер + «+N страницы». **Режим «проценты» отменён пользователем** (был в плане v1, решили убрать как лишний).
- Колокол напоминания — в стрик-карточке; шапка Home разгружена.
- Тёмная тема всех новых блоков корректна.

### Полировка, перенесённая из дизайна в Compose (не гоняем через Claude design ещё раз)
Чек-лист на время верстки — мелкие визуальные доделки, дешевле сделать в коде:
- [ ] Обложки: плоские матовые, типографский плейсхолдер без дефис-переносов («Нор-веж-ский»).
- [ ] Карточки библиотеки: одинаковая высота, заголовок clamp в 2 строки.
- [ ] Инпуты: тёплые `#FBF6EF` вместо чисто-белых; убрать дефолт «0» → placeholder.
- [ ] Кебаб `⋮` на hero-card убрать (быстрые чипы делают его лишним).
- [ ] «Удалить книгу» в Edit — видимая отдельная кнопка, не под фолдом.
- [ ] Прогресс: воздух между заголовком и «Дочитал до страницы».
- [ ] Вторичный текст притемнить на caption-уровне; единый stroke-вес иконок.

- Заморозка стрика, поиск/сортировка в библиотеке — будущее.
