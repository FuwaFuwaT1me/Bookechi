# Bookechi — описание работы приложения

> Спутник для чтения: тёплый, минималистичный трекер прогресса по книгам.
> Документ описывает **что приложение делает и как себя ведёт** (для дизайна и дальнейшей разработки).

---

## 1. Суть и пользователь

**Кому:** человеку, который читает книги и хочет видеть прогресс и не терять ритм.
**Обещание:** открыл приложение → увидел текущую книгу → за пару тапов отметил, до какой страницы дочитал сегодня → получил тёплый фидбэк и сохранил серию.

**Главный цикл (core loop):**
```
открыл Home → активная книга в фокусе → «Отметить прогресс»
   → ввёл страницу остановки → авторасчёт «прочитано сегодня»
   → Success: стрик +1, % прогресса вырос → «Готово» → назад на Home
```

---

## 2. Доменная модель

- **Book** — `name, author, обложка, pages (всего), currentPage (текущая), status, isFavorite`.
- **Статусы (в UI):** `В планах` · `Читаю` · `Прочитано`. «Избранное» — независимый флаг (♥), не статус.
- **ReadingSession** — одна запись = один день чтения одной книги: `date, startPage, endPage, pagesRead (=endPage−startPage), readingTimeMinutes (опц.)`.
- **Активная книга** — книга со статусом `Читаю`; если их несколько — та, по которой последний прогресс.

---

## 3. Карта экранов

```
┌─ Bottom Nav ───────────────────────────────────────────┐
│  [ Активность ]   [ Продуктивность ]   [ Библиотека ]   │
└─────────────────────────────────────────────────────────┘
        │                  │                    │
     Home/Activity       Stats               Library
        │                                        │
        └── «Отметить прогресс» → Progress → Success
        └── (из Library/Home) → Add book / Edit book
```

10 требуемых экранов = 3 таба (с состояниями) + 4 потоковых экрана.

---

## 4. Экраны и поведение

### 4.1 Home / Активность
**Назначение:** дать мгновенный фокус на текущей книге и быстрый ввод прогресса.

Содержимое (наполненное состояние):
- **Приветствие** — «Добрый вечер» + имя/нейтрально, лёгкий подзаголовок.
- **Блок стрика** — серия дней подряд (огонёк + число), мини-полоска последних дней.
- **Hero-card активной книги** — обложка, название, автор, `currentPage / pages`, progress bar, процент, **прогноз дочитывания** («при твоём темпе ~6 дней до конца», из ср. стр/день), **CTA «Отметить прогресс»**.
- **Ниже — компактный список** остальных книг (читаю/в планах): обложка-миниатюра, название, автор, мини-прогресс.

**Состояния Home:**
- (a) **нет книг вообще** → тёплый empty-state: «Добавьте первую книгу» + CTA.
- (b) **книги есть, активной нет** → подсказка «Выберите, что читаете сейчас» + список книг в планах.
- (c) **активная есть, прогресс сегодня не отмечен** → hero-card с ненавязчивым нуджем «отметьте сегодняшние страницы».
- (d) **активная есть, прогресс отмечен** → hero-card с обновлённым прогрессом + спокойное «на сегодня готово».

### 4.2 Stats / Продуктивность
**Назначение:** показать ритм и динамику чтения.

- **4 карточки метрик:** Книг прочитано · Страниц прочитано · Дней без перерывов · Стр./день в среднем.
- **Переключатель периода:** Месяц / Год (segmented).
- **Heatmap активности** — сетка дней, интенсивность = страниц за день (6 уровней).
- **Итоговая карточка периода** — сводка с акцентом (всего страниц/книг за период, лучший день/серия).
- **Zero-state:** когда данных нет — мягкая заглушка «Здесь появится ваша статистика, как только отметите первый прогресс».

**Определения метрик** (за выбранный период): см. ADR-3.7 — книг завершено; `Σ pagesRead`; макс. цепочка дней с чтением; `Σ pagesRead / активных дней`.
- **Скорость чтения** (стр/час) — доп-метрика, показывается только если пользователь вёл время чтения.

### 4.3 Library / Библиотека
**Назначение:** управлять коллекцией книг.

- **Заголовок + количество** книг.
- **Фильтры:** Все · В планах · Читаю · Прочитано · Любимое.
- **Список/сетка карточек:** обложка, название, автор, статус-чип, ♥, мини-прогресс для «читаю».
- **FAB** — добавить книгу.
- **Empty-state:** «Библиотека пуста» + CTA «Добавить книгу».
- Тап по карточке → детали/редактирование.

### 4.4 Add book / Добавить книгу
- Тёплый, не «холодный form-look».
- **Блок обложки** (загрузка/плейсхолдер).
- Поля: **Название**, **Автор**, **Всего страниц**.
- **Выбор статуса:** Читаю / В планах / Прочитано (сегментировано/чипами).
- Если статус **«Читаю»** → появляется поле **Текущая страница**.
- Один **главный CTA** «Добавить книгу».
- Валидации: название не пустое; страниц > 0; текущая ≤ всего.

### 4.5 Edit book / Редактировать книгу
- Та же форма, предзаполнена.
- Доп. действия: переключить ♥, сменить статус, удалить книгу (с подтверждением).
- CTA «Сохранить».

### 4.6 Progress / Отметить прогресс
- Сверху — **книга** (обложка, название, прогресс «было»).
- **Стартовая страница** (откуда продолжаем = `currentPage`).
- **Крупное поле ввода** — страница, на которой остановился сегодня; справа — **total pages**.
- **Авторасчёт** «прочитано сегодня» = `введённая − стартовая` (живой пересчёт).
- Опционально — **время чтения** (мин).
- Валидации: значение > стартовой и ≤ total; иначе мягкая ошибка.
- CTA «Сохранить прогресс».

### 4.7 Success / Прогресс сохранён
- **Стрик** (серия дней, +1 если сегодня впервые).
- **Сколько страниц прочитано сегодня**.
- **Изменение процента** прогресса (было → стало).
- **Мотивирующий, но сдержанный** фидбэк (одна тёплая фраза).
- Кнопка **«Готово»** → Home.
- Вариант «книга дочитана» (если `endPage == pages`): особый тёплый финал «Книга прочитана», авто-перевод в «Прочитано» + опциональные **оценка 1–5 ★** и **короткая заметка/цитата** (задел под «полку прочитанного»).

---

## 5. Ключевые правила и edge-cases

- После сохранения прогресса: `Book.currentPage = endPage`; обновляются стрик и Stats.
- Стрик считается по дням с `pagesRead > 0`; пропущенный день обрывает серию.
- Нельзя ввести страницу меньше стартовой или больше total — мягкая валидация.
- Удаление книги каскадно убирает её `ReadingSession` (уже задано в Room через `ForeignKey CASCADE`).
- Время чтения опционально; метрики на его основе показываются только когда оно заполнено.

## 6. Тон и визуальный язык (кратко)

Палитра «Terracotta & Linen»: льняная бумага, ореховая типографика (serif-заголовки + sans-тело),
приглушённая терракота как акцент и шалфей как второй акцент, мягкие радиусы, много воздуха,
одна чёткая иерархия на экран, без перегруза. Подробные токены — в `docs/design-prompt.md`.
