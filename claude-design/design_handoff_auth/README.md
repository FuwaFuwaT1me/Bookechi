# Промпт для Claude Code — экран авторизации Bookechi

> Скопируй всё, что ниже разделителя, в Claude Code. В корне бандла лежат файлы-референсы
> (`Bookechi — Авторизация.html` + `bookechi-auth/`, `bookechi/`, `components/`) — открой их в браузере,
> чтобы видеть поведение и точные значения. Это **дизайн-референс на HTML, а не продакшн-код для копирования.**

---

## Задача

Реализуй экран авторизации мобильного приложения **Bookechi** (тёплый дневник чтения, RU-интерфейс)
в нашем кодовом окружении, **повторив дизайн пиксель-в-пиксель** и используя наши существующие
паттерны/компоненты/библиотеки. Если фреймворк ещё не выбран — возьми наиболее подходящий
(референс написан на React + чистый CSS, легко ложится на React Native / SwiftUI / Flutter).

Фиделити: **high-fidelity.** Цвета, типографика, отступы, копирайт и анимации в референсе — финальные.

В бандле есть 4 варианта приветственного экрана (переключаются Tweaks-панелью — это была дизайн-разведка,
**в проде Tweaks не нужен**). **В прод идёт вариант «Обложки» (полка с книгами)** как дефолт.
Остальные три (`Тёплый минимал`, `Редакторский`, `Дуга`) — задокументированы ниже на случай A/B,
но реализовывать обязателен только «Обложки».

## Три способа входа (иерархия неизменна)

1. **Google** — главная кнопка (самая заметная, filled-terracotta).
2. **Почта/пароль** — вторичная (outline). Включает вход, регистрацию и восстановление пароля.
3. **Анонимно** — третичная (текстовая ссылка). Перед входом показывает шторку с честными ограничениями.

## Карта экранов и состояний (state-машина)

```
welcome (база)
 ├─ Google      → connecting (спиннер ~1.1с) → success(via=google)
 ├─ Почта       → email-push
 │                 ├─ signin   → (валидация, ~0.7с) → success(via=email)
 │                 ├─ register → (валидация, ~0.7с) → success(via=email)
 │                 ├─ recover  → (валидация, ~0.6с) → sent
 │                 └─ Google внутри формы → connecting → success
 └─ Анонимно    → anon-sheet
                   ├─ «Продолжить без аккаунта» → success(via=anon)
                   └─ «Лучше создать аккаунт»   → email-push(register)

success → «Открыть библиотеку» (роут на главный экран приложения)
        → «Сменить способ входа» (reset → welcome)   [для anon: «Создать аккаунт» → email register]
```

Состояние: `flow ∈ {null, email, anon, connecting, done}`, `via ∈ {google, email, anon}`,
`emailMode ∈ {signin, register, recover, sent}`. В референсе — `bookechi-auth/auth.jsx`.

## Экран 1 — Welcome «Обложки» (главный)

Вертикальная раскладка, центр по вертикали, паддинг `0 24px`.

- **Герой:** веер из 3 «обложек» книг на мягком радиальном фоне
  (`radial-gradient(115% 78% at 50% 30%, var(--accent-soft), transparent 70%)`, скруглён `--r-hero`).
  Обложки — **public-domain классика** (без проблем с авторским правом):
  - левая, наклон −15°, смещение (−58, 6), цвет спинки `#7C8A6E` / текст `#FBF6EF`: **«Анна Каренина» · Л. Толстой**
  - правая, наклон +15°, смещение (58, 6), `#9E4A2C` / `#FFF1E6`: **«Преступление и наказание» · Ф. Достоевский**
  - центральная (поверх, z3), наклон 0°, `#BE5E3B` / `#FFF6EE`: **«Евгений Онегин» · А. Пушкин**
  - каждая обложка: ширина 92, скругление 11, высота = 92×1.45, лёгкая тень
    `drop-shadow(0 16px 26px rgba(56,42,32,0.34))`, вертикальная «прошивка» корешка слева,
    название — Lora 12.5px/600 (clamp 4 строки), автор — 9px uppercase. См. `BookCover` в `bookechi/ui.jsx`.
- **Заголовок:** «Ваша полка ждёт» — Lora 600, 29px.
- **Подзаголовок:** «Соберите книги, которые читаете и любите — Bookechi сохранит каждую страницу.» —
  Inter 15.5px, `var(--text2)`, max-width 272, `text-wrap: pretty`.
- **Кнопки (снизу, стек, gap 12):**
  - Google — `Продолжить с Google`
  - Почта — `Войти по почте`
  - Аноним — `Сначала осмотреться` (текст + шеврон →)
- **Легал:** «Продолжая, вы соглашаетесь с Условиями и Политикой конфиденциальности.»
  — 12px, центр, ссылки `var(--accent-deep)`/600.

### Кнопки методов (общий компонент, `AuthActions`)
- **Google (главная, filled):** `min-height 54`, скругление `--r-btn`, фон `var(--accent)`,
  текст `var(--on-accent)` 16px/600; слева в кружке Ø30 белый чип с **официальным 4-цветным «G»**
  (SVG в `auth-welcome.jsx → GoogleG`); `:active` → фон `var(--accent-pressed)`, `scale(.98)`.
- **Почта (вторичная, outline):** фон `var(--surface-pure)` (в тёмной — `var(--surface)`),
  бордер `1.5px var(--stroke)`, текст `var(--text)`; слева линейная иконка конверта.
- **Аноним (третичная, текст):** `min-height 46`, текст `var(--text2)` 15px/600 + шеврон,
  `:active` → текст `var(--text)`, шеврон сдвигается на 3px.
- Порядок при `primary=Google`: Google → Почта → Аноним. (Тумблер «главная кнопка» в Tweaks
  умеет менять Google↔Почта местами и менять заливку — в проде это не нужно, оставь Google главной.)

## Экран 2 — Почта (push поверх welcome, слайд справа)

Шапка: круглая кнопка «назад» (←) + заголовок (Lora 21px) + мини-марка приложения справа.
Кнопка назад: из `recover`/`sent` → возвращает в `signin`; из `signin`/`register` → закрывает push.

**Вход / Регистрация** — сегмент-контрол (`bk-seg`: pill, фон `var(--chip-bg)`, активный таб —
`var(--surface-pure)` с тенью):
- **Поля** (`bk-input`: высота 52, скругление 16, бордер `1.5px var(--stroke)`, фокус — бордер `var(--accent)`,
  ошибка — бордер `var(--error)` + лёгкий красный фон):
  - Регистрация добавляет поле **«Имя»** (плейсхолдер «Как к вам обращаться»).
  - **«Электронная почта»** (type=email, плейсхолдер `you@example.com`).
  - **«Пароль»** — с кнопкой показать/скрыть (иконка глаза справа). В регистрации хинт «Минимум 6 символов».
- В режиме `signin` справа над кнопкой — ссылка **«Забыли пароль?»** → `recover`.
- **Главная кнопка:** `Войти` / `Создать аккаунт` (filled-terracotta). На время запроса — крутящийся
  спиннер (бордер-кольцо `var(--on-accent)`, 0.7с linear).
- В регистрации под кнопкой — легал-строка.
- Разделитель **«или»** (линии + лейбл `var(--text2)`).
- **`Продолжить с Google`** (outline) — тот же Google-флоу.
- Футер-подсказка: «Ещё нет аккаунта? **Создать**» / «Уже есть аккаунт? **Войти**».

**Восстановление (`recover`):** текст «Укажите почту от аккаунта — пришлём ссылку, чтобы задать новый пароль.»,
поле почты, кнопка **`Отправить ссылку`**, ссылка «Вернуться ко входу».

**Письмо отправлено (`sent`):** иконка конверта в плашке `var(--accent-soft)`, заголовок «Проверьте почту»,
текст «Отправили ссылку для сброса на **{email}**. Перейдите по ней, чтобы задать новый пароль.»,
кнопка `Открыть почтовое приложение`, ссылка «Вернуться ко входу».

**Валидация:** email — `/^\S+@\S+\.\S+$/` («Проверьте адрес — кажется, есть опечатка»);
пароль — длина ≥ 6 («Минимум 6 символов»). Реф: `bookechi-auth/auth-flows.jsx → EmailScreen`.

## Экран 3 — Анонимно (bottom sheet, слайд снизу + затемнение)

- Хедер: иконка телефона в плашке `var(--accent-soft)` + «Зайти без аккаунта» +
  «Можно начать прямо сейчас — вот что важно знать».
- **Три строки-ограничения** (иконка-плашка 38×38 + заголовок 14.5/600 + подпись-caption):
  - ⚠ (фон `var(--error-bg)`, цвет `var(--error)`) **«Нет синхронизации и бэкапа»** —
    «Книги и серия чтения хранятся только на этом телефоне.»
  - ⚠ **«Прогресс не перенесётся»** — «При удалении приложения или смене устройства данные потеряются.»
  - ✓ (фон `var(--sage-soft)`, цвет `var(--sage)`) **«Аккаунт можно создать позже»** —
    «Всё, что уже прочитано, сохранится и переедет в облако.»
  - между строками — разделитель `1px var(--divider)`.
- Кнопки: **`Продолжить без аккаунта`** (filled) + **`Лучше создать аккаунт`** (`bk-btn-secondary`,
  ведёт в форму почты в режиме регистрации).

## Экран 4 — Connecting (Google)

Полноэкранный fade на `var(--canvas)`: спиннер Ø64 (акцентное кольцо) с 4-цветным «G» в центре,
«Подключаем Google…» (Lora 19px) + «Секунду — открываем безопасный вход». В проде заменить
на реальный редирект/попап Google OAuth.

## Экран 5 — Success / boot

Fade на `var(--canvas)`, центр: «печать» Ø88 (градиент `--accent`→`--accent-deep`) с галочкой
(анимация `bk-pop` — scale 0.45→1.12→1), заголовок Lora 30px + подзаголовок. Копирайт по способу:
- google → «Вы вошли» / «Аккаунт Google подключён. Рады видеть вас в Bookechi.»
- email → «Добро пожаловать» / «Аккаунт готов — ваша полка ждёт первую отметку.»
- anon → «Готово» / «Вы зашли без аккаунта. Прогресс хранится на этом устройстве.» +
  плашка-чип «Данные на этом устройстве».
- Кнопки: **`Открыть библиотеку`** (filled, роут на главный экран) + текстовая «Сменить способ входа»
  (для anon — «Создать аккаунт»).

## Анимации

- **Push (форма почты):** `transform: translateX(100%→0)`, `0.34s cubic-bezier(.32,.72,.28,1)`.
- **Sheet (аноним):** `translateY(105%→0)`, `0.38s cubic-bezier(.32,.72,.28,1)`; скрим `opacity 0.3s`.
- **Fade (connecting/success):** `opacity`, `0.3–0.4s`.
- **Вход контента welcome:** `translateY(16px→0)`, `0.55s`, со степ-задержками 0/0.08/0.16с
  (только transform — чтобы не прятать контент при статичном рендере/скриншоте; уважать `prefers-reduced-motion`).
- **Спиннер:** вращение 0.7s linear; при reduced-motion — пульс прозрачности.
- Хит-таргеты ≥ 44px, видимый фокус на полях, `aria-label` на иконочных кнопках.

## Что подключить на бэкенде (прототип это имитирует)

- **Google** — реальный OAuth (вместо фейкового «connecting ~1.1с»).
- **Почта** — вход, регистрация, отправка письма для сброса пароля (вместо фейковых таймаутов).
- **Аноним** — гостевая сессия, данные только локально, без синка; апселл «создать аккаунт» — апгрейд гостя
  с переносом данных.
- «Открыть библиотеку» — навигация на главный экран приложения.

## Ассеты

Внешних ассетов нет: марка приложения (стопка книг) и логотип Google — инлайн-SVG в `auth-welcome.jsx`
(`StackGlyph`, `BrandMark`, `GoogleG`). Иконки — линейный 24px-сет (`BkIcon` в `ui.jsx`, `AuthIcon` в
`auth-welcome.jsx`). Шрифты Lora + Inter — Google Fonts.
